# Компьютерный тренажерный комплекс ЭЛОУ-АВТ (ИИ-модуль)

Интеллектуальный модуль в составе Компьютерного тренажерного комплекса (КТК) для обучения операторов технологической установки первичной переработки нефти **ЭЛОУ-АВТ**.

Проект разрабатывается в рамках IT-чемпионата нефтяной отрасли («Энерготехнохаб Петербург» / CASE-IN).

## 🎮 Что тестировать и как (Функционал КТК)

Компьютерный тренажерный комплекс содержит два независимых веб-интерфейса (К1: Разделение экранов) с передачей телеметрии и команд по WebSockets в реальном времени.

### 1. Панель Оператора (http://localhost:5173/)
* **Мнемосхема SCADA:** Отображает динамические датчики: Температура печи П-1 (T-1), Давление в колонне К-1 (P-1), Уровень куба К-1 (L-1).
* **Органы управления:**
  - Переключение клапанов: Вход сырья V-1, Аварийный сброс V-2, Дренаж куба V-3.
  - Управление нагревом: Ползунок уставки температуры печи П-1.
  - Кнопка аварийного останова (ESD) для глушения установки.
* **ИИ-Мониторинг:**
  - Живой график предиктивного тренда параметров на 15 секунд вперед.
  - Индикатор риска аварии (Risk Level, %) на основе LSTM-модели.
  - Персональные всплывающие предупреждения ИИ по регламенту.

### 2. Панель Инструктора (http://localhost:5173/ с переключением роли)
* **Управление сценарием:** Выбор пуска/останова и сброс сессии.
* **Инъекция неисправностей (Слайд 11 КТК):**
  - **Отказ сырьевого насоса Н-1** (прекращает подачу сырья, угроза сухого хода П-1).
  - **Прогар змеевика П-1** (критический перегрев змеевиков, риск пожара).
  - **Зависание клапана V-2** (блокировка сброса давления в факел, угроза взрыва).
* **Аудит обучения и ИБ:**
  - База данных сессий с проверкой контрольной суммы результатов по ГОСТ (хэширование SHA-256 с солью для исключения фальсификации).
  - **Интерактивный разбор:** Кликните по любой строке в таблице сессий — откроется центрированное модальное окно с детальным ScoreCard (оценка, точечные нарушения регламента и рекомендации ИИ) и полным посекундным логом действий оператора за эту попытку.

## 🧪 Сценарии для тестирования

### Тест-кейс 1. Успешный пуск по регламенту (Score: 100%, Grade: A)
1. Откройте сырьевой клапан V-1.
2. Плавно поднимите температуру печи П-1 (передвиньте ползунок уставки до ~310°C).
3. При росте уровня куба откройте дренаж V-3 для балансировки системы.
4. Завершите сессию. Оценка ИИ будет 100%.

### Тест-кейс 2. Парирование отказа сырьевого насоса
1. Инструктор включает тумблер «Отказ насоса Н-1».
2. На панели оператора падает расход холодного сырья, змеевики начинают опасно греться.
3. Оператор обязан немедленно снизить уставку температуры печи П-1 (SP_DOWN) до минимума для тушения топки.
4. Если оператор успел снизить температуру — ИИ засчитает успешную локализацию нештатной ситуации (100% / Grade A). Если нет — оценка упадет из-за нарушения п. 7.9.1 (нагрев всухую).

### Тест-кейс 3. Авария при заклинившем клапане сброса V-2
1. Инструктор включает «Зависание клапана V-2» и поднимает температуру.
2. Давление в колонне К-1 начинает расти. Оператор пытается открыть V-2, но клапан не реагирует (блокировка).
3. Оператор обязан запустить ручной аварийный останов (ESD). ИИ оценит действия оператора по DTW с учетом непреодолимого дефекта.
4. Если оператор бездействует — произойдет авария (взрыв К-1 при >0.48 МПа) и оценка упадет до 0% / Grade F.

### Тест-кейс 4. Контроль целостности ИБ (К8)
1. База сессий инструктора проверяет целостность данных. При корректных записях в графе ИБ-контроль горит зеленый значок `OK`.
2. Попробуйте открыть файл базы данных `backend/tutor.db` через SQLite редактор и вручную изменить оценку `score` или текст нарушений в таблице `training_sessions`.
3. При обновлении истории инструктора система обнаружит несовпадение SHA-256 хэша и выведет красное предупреждение `Нарушена!`.

---

*   📂 **`frontend/`** — интерактивный веб-интерфейс тренажера (React + Vite + TypeScript).
*   📂 **`backend/`** — API-сервер для связи интерфейса с симулятором и ИИ (FastAPI / Python).
*   📂 **`ai_core/`** — код ИИ-модели для классификации и прогнозирования ошибок оператора.
*   📂 **`simulator/`** — математическая модель технологических процессов установки ЭЛОУ-АВТ.
*   📂 **`docs/`** — документация по проекту (экономические расчеты, модель угроз ИБ).

## 🚀 Быстрый старт

### Вариант 1: Единый запуск (фронтенд + бэкенд одновременно)
Самый простой способ запустить всё одной командой из корня проекта:
```bash
# 1. Установите Python-зависимости (если еще не установлены)
pip install --user -r backend/requirements.txt

# 2. Установите npm-зависимости во фронтенде (если еще не установлены)
cd frontend && npm install && cd ..

# 3. Запустите единую команду в корне проекта
npm run dev
```
Интерфейс откроется по адресу: http://localhost:5173/ (или http://127.0.0.1:5173/)
API бэкенда будет работать на: http://127.0.0.1:8000/

### Вариант 2: Раздельный локальный запуск

Все команды выполняются из корня проекта:

#### 1. Запуск только бэкенда (API и Симулятор)
```bash
npm run backend
```
*(Или напрямую: `python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000`)*

#### 2. Запуск только фронтенда (Интерфейс)
```bash
npm run frontend
```
*(Или напрямую: `cd frontend && npm run dev`)*

---

### Вариант 3: Запуск через Docker Compose (всё в одном)

Для сборки и запуска всего стека (бэкенд на порту `8000`, фронтенд на порту `80`):
```bash
docker-compose up --build
```
