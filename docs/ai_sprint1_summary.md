# 🤖 Резюме обзора ИИ-технологий для MVP (Спринт 1)

Этот файл подготовлен для демонстрации и обсуждения на созвоне команды. Он содержит краткую выжимку по выбранным технологиям ИИ для Компьютерного тренажерного комплекса (КТК) установки ЭЛОУ-АВТ.

---

## 🎯 Ключевые требования для оценки 5 баллов
Чтобы интеллектуальный модуль получил наивысший балл, мы должны реализовать:
1. **Прогноз аварийных рисков** до совершения критических ошибок.
2. **Локализацию ошибок оператора во времени** с объяснением причинно-следственных связей.
3. **Персональные рекомендации** и адаптивные сценарии обучения.

---

## 🏗 Архитектура ИИ-модуля (Гибридный подход)

Мы внедряем две независимые модели на бэкенде (FastAPI), взаимодействующие с фронтендом через WebSockets в реальном времени:

### 1. Блок прогнозирования рисков (Predictive Engine)
* **Модель:** Сеть **LSTM / GRU**, экспортированная в формат **ONNX** (для высокой производительности на CPU без необходимости устанавливать PyTorch).
* **Принцип:** 
  - Каждую секунду фронтенд отправляет скользящее окно показаний приборов (за последние 30 секунд: температура, давление, уровень).
  - LSTM прогнозирует тренд параметров на 15 секунд вперед.
  - Если прогнозируемые значения выходят за рамки безопасного диапазона (например, температура > 310°C), рассчитывается процент риска аварии.
* **Результат в UI:** Индикатор уровня риска на SCADA-панели предупреждает оператора до наступления аварии.

### 2. Блок поиска ошибок регламента (Error Analyzer)
* **Алгоритм:** **DTW (Dynamic Time Warping)** + База знаний регламента в SQLite/JSON.
* **Принцип:**
  - Симулятор записывает последовательность кликов и переключений задвижек оператора.
  - DTW сравнивает этот вектор с эталонным («золотым стандартом») сценарием пуска/останова.
  - При отклонении алгоритм находит шаг, где совершена ошибка, и достает соответствующее правило из базы регламента.
* **Результат в UI:** Текстовая подсказка в журнале (например: *«Ошибка: Нарушена последовательность. Вы перекрыли дренаж V-3, не открыв сброс давления V-2...»*).

### 3. Перспектива: LLM + RAG (Локальный ИИ-ассистент)
* **Стек:** Локальная модель **Llama-3-8B-Instruct-RU** (через Ollama) + векторная БД **ChromaDB**.
* **Принцип:** База содержит текст технологического регламента ЭЛОУ-АВТ. При ошибке LLM генерирует развернутое объяснение нарушения на основе извлеченного пункта регламента.
