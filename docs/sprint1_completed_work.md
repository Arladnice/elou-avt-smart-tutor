# 🏆 Отчет о результатах Спринта 1 (02.07 — 09.07)
## Компьютерный тренажерный комплекс ЭЛОУ-АВТ (Smart Tutor)

В рамках первого спринта команда успешно завершила фазу исследований, спроектировала архитектуру системы и заложила программный фундамент для разработки MVP. Ниже приведен детальный перечень выполненных работ по направлениям и участникам.

## 💻 1. Разработка и окружение (Денис)
* **Создание Monorepo структуры:** Репозиторий разделен на изолированные модули: `frontend/` (React + TS), `backend/` (FastAPI), `ai_core/` (ИИ-движок) и `simulator/` (физ-хим модель).
* **Унификация запуска (Root package.json):** 
  - Создан корневой [package.json](file:///c:/Projects/elou-avt-smart-tutor/package.json), позволяющий запускать весь комплекс (фронтенд + бэкенд) одной командой `npm run dev`.
  - Добавлены скрипты раздельного запуска из корня: `npm run backend` и `npm run frontend`.
* **Оптимизация сетевых биндингов:** 
  - Конфигурация Vite во фронтенде ([vite.config.ts](file:///c:/Projects/elou-avt-smart-tutor/frontend/vite.config.ts)) переведена на хост `0.0.0.0`, что предотвратило ошибки `ERR_CONNECTION_REFUSED` при локальном доступе по IPv4.
* **Переносимость рабочего окружения:**
  - В правилах разработки ([.agents/AGENTS.md](file:///c:/Projects/elou-avt-smart-tutor/.agents/AGENTS.md)) все абсолютные пути заменены на относительные, благодаря чему правила ИИ-ассистента теперь корректно работают как на рабочих, так и на домашних станциях участников.
* **Документирование быстрого старта:** 
  - Полностью переписан корневой [README.md](file:///c:/Projects/elou-avt-smart-tutor/README.md) с описанием локального запуска и запуска в Docker.

## ⚙️ 3. Физико-химическое моделирование (Андрей)
* **Математическая модель процесса:** Разработана базовая модель блоков электрообессоливания (ЭЛОУ) и атмосферной перегонки (АТ) на Python в модуле `simulator/elou_avt_model.py`.
* **Параметризация:** Заданы переменные датчиков (температура печи П-1, давление в колонне К-1, уровень куба К-1) и регулирующих клапанов (сырье, топливный газ, орошение, дренаж).
* **Аварийные уставки:** В коде прописана логика выхода параметров за технологические пределы по техрегламенту (перегрев печи > 310°C, избыточное давление > 0.4 МПа).
