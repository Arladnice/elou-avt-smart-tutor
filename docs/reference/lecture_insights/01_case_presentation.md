## Ключевые требования кейса

### Техническая реализация КТК
1. **КТК = Компьютерный тренажёрный комплекс** для обучения операторов ЭЛОУ-АВТ
2. **Два уровня:** физическая модель техпроцесса + сценарии дефектов (неисправностей)
3. **АРМ Оператора (SCADA):** мнемосхема, тренды, аварийные уставки
4. **АРМ Инструктора:** управление сценариями, инъекция дефектов, оценка действий оператора
5. **ИИ-модуль:** предиктивная аналитика (прогноз рисков), подсказки оператору

### Что требуют спикеры
| Требование | Цитата | Статус в проекте |
|---|---|---|
| Нет реальной телеметрии — нужна синтетика | «Телеметрию мы предоставлять не можем» | ✅ Реализовано: `simulator/` генерирует синтетику |
| Осторожно с тяжёлыми LLM | «Требуются большие ресурсы и ограничения по лицензиям» | ✅ Отказались от LLM в пользу локальной базы знаний |
| Масштабируемость под другие установки | «Ваше решение должно масштабироваться» | ⚠️ Нужен JSON-конфиг описания установки |
| Сценарии дефектов из коробки | «Второй уровень — формализация типовых дефектов» | ✅ Реализовано 5 сценариев |
| Тренды и инженерная интуиция | «Опытный оператор по мельчайшим колебаниям определяет» | ⚠️ Нужны sparklines для $dT/dt$, $dP/dt$ |
| Артефакты для дедлайна 04.08 | Презентация + пояснительный документ + видеодемо | ⚠️ В процессе |
