## Основные тезисы

### 1. Структура экономического обоснования проекта
Обязательные элементы презентации для инвестиционного комитета:
1. **Описание проблемы** (зачем делаем проект)
2. **Описание решения** (функционал, интеграция, техническая составляющая)
3. **Итоговый образ результата** (законченная формулировка, что получим)
4. **Ожидаемые экономические эффекты** (измеримые в деньгах)
5. **Нематериальные эффекты** (безопасность, снижение рисков)
6. **Совокупная стоимость владения** (CAPEX + OPEX + сервисные затраты)
7. **Продуктовые метрики** (количество пользователей, частота использования)
8. **Оценка рисков**

### 2. Процесс оценки эффективности: As-Is → To-Be → Эффект
1. Описать бизнес-процесс **As-Is** по экономическим статьям затрат (минуты, часы, количество операций)
2. Описать бизнес-процесс **To-Be** (на основе бенчмарка или экспертной оценки)
3. Определить **драйверы эффективности** (KPI)
4. Рассчитать **экономический эффект** = стоимость As-Is − стоимость To-Be
5. Оценить в денежном выражении по годам (горизонт 5 лет)

### 3. Три типа экономических эффектов
| Тип | Описание | Пример для КТК |
|---|---|---|
| **Прямой (NPV)** | Измеримый, прямое влияние на денежный поток | Сокращение персонала инструкторов (маловероятно) |
| **Косвенный (EMV)** | Измеримый, но нет прямого влияния на ден. поток | Сокращение времени обучения оператора |
| **Трудноизмеримый (DMV)** | Экспертная оценка, сложно подтвердить | Снижение рисков аварий, экологических штрафов |

### 4. Совокупная стоимость владения (TCO)
- **CAPEX:** разработка, тестирование, оборудование, лицензии
- **OPEX:** аренда инфраструктуры, документация, сопутствующие затраты
- **Сервисные затраты:** поддержка после вывода в промышленную эксплуатацию (лицензии, обслуживание)

> *«Вас обязательно спросят на защите: а где затраты на обслуживание всего добра, которое вы планируете разработать»*

### 5. NPV, дисконтирование, WACC
- Горизонт оценки: **5 лет** от ввода в промышленную эксплуатацию
- Формула дисконтирования: `DCF = CF / (1 + i)^t`
- **NPV > 0** = проект окупается и приносит прибыль сверх требуемой доходности
- Ставка дисконтирования ≈ WACC (средневзвешенная стоимость капитала)

## Применение к нашему проекту КТК ЭЛОУ-АВТ

### Описание проблемы (As-Is)
- Обучение нового оператора ЭЛОУ-АВТ занимает 6–12 месяцев наставничества
- Нет возможности безопасно отработать аварийные сценарии на реальной установке
- Стоимость простоя ЭЛОУ-АВТ при реальном инциденте: ~$50K–$500K в зависимости от масштаба
- Человеческий фактор — основная причина ~60% инцидентов на НПЗ

### Образ результата (To-Be)
- КТК позволяет обучить оператора за 2–3 месяца вместо 6–12
- 100% операторов проходят отработку аварийных сценариев в безопасной среде
- ИИ-модуль подсвечивает ранние признаки аварий, формируя «инженерную интуицию»

### Дерево KPI проекта
```
Бизнес-эффект
├── Сокращение времени обучения оператора
│   ├── As-Is: 6 мес × стоимость наставника (120 чел-дней × 5000₽/день = 600K₽)
│   └── To-Be: 3 мес × (стоимость тренажёра/5 операторов) ≈ 100K₽ per capita
│
├── Снижение инцидентов (трудноизмеримый)
│   ├── As-Is: ~3 инцидента/год с ущербом ~5M₽ каждый = 15M₽/год
│   └── To-Be: 20% снижение → экономия ~3M₽/год
│
└── Снижение простоев при обучении на реальной установке
    ├── As-Is: 10 дней/год простой для тренировок
    └── To-Be: 0 дней (тренировки на КТК)
```

### Затраты проекта (оценка)
| Статья | Сумма (экспертная) |
|---|---|
| CAPEX: Разработка КТК (команда 4 чел × 3 мес) | ~2.4M₽ |
| CAPEX: Сервер / рабочая станция | ~300K₽ |
| OPEX: Документация, тестирование | ~200K₽ |
| Сервисные: Поддержка и обновление (1 год) | ~500K₽/год |
| **Итого (Year 0)** | **~3.4M₽** |
