# КТК ЭЛОУ-АВТ Smart Tutor — Backend API

API-сервер реального времени для связи веб-интерфейса тренажера с физико-математическим симулятором процессов ЭЛОУ-АВТ и ИИ-модулем оценки оператора.

---

## 🎯 Стек технологий

* **Python 3.11+** — среда выполнения.
* **FastAPI** — веб-фреймворк для REST API и WebSocket.
* **Uvicorn** — ASGI-сервер.
* **SQLite3** — легкая встроенная база данных для хранения результатов сессий и журналов ИБ.
* **ONNX Runtime** — выполнение инференса нейросети RiskLSTM в реальном времени.

---

## 📂 Модульная структура бэкенда

Бэкенд строго следует стандартам модульности и безопасности проекта:

```
backend/
├── main.py                    # Инициализация FastAPI приложения и lifespan события
├── routes/                    # Эндпоинты REST API и WebSocket
│   ├── auth.py                # POST /api/auth/login (авторизация по роли)
│   ├── health.py              # GET /api/health (легковесный health-check для keep-alive)
│   ├── sessions.py            # GET /api/sessions (история), POST /api/sessions/clear
│   └── ws.py                  # WebSocket /ws (обмен данными симуляции)
├── services/                  # Бизнес-логика симуляции и сокетов
│   ├── connection_manager.py  # Агрегатор состояния, рассылка сокетов, сохранение сессий
│   └── simulation_loop.py     # Фоновый поток циклического шага физики и ТБ-проверок
├── models/                    # Валидация Pydantic v2
│   └── schemas.py             # Схемы запросов и ответов
├── db/                        # Слой работы с базой данных
│   ├── database.py            # Настройка подключения к SQLite и init_db()
│   └── queries.py             # SQL-запросы истории и результатов
└── utils/                     # Вспомогательные скрипты
    ├── security.py            # Хэширование целостности (SHA-256) и аудит-логирование ИБ
    └── helpers.py             # Генераторы ID и хелперы
```

---

## 🔒 Безопасность и контроль целостности (К8: ИБ)

1. **Защита от НСД**: Доступ к истории тренировок разграничен.
2. **Контроль целостности записей**: Каждая сессия в таблице `training_sessions` подписывается криптографическим хэшем SHA-256 с секретной солью, включающим в расчет все параметры (время, оценку, ошибки) и полные логи действий оператора. Любая попытка подделать оценку или логи напрямую в БД мгновенно выявляется при чтении API.
3. **Журнал аудита**: Все значимые события (авторизация, управление задвижками, инжекция дефектов, сброс базы данных) записываются в защищенную таблицу `audit_logs` с расчетом хэша целостности.

## 🚀 Быстрый запуск

### 1. Установка зависимостей
Установите библиотеки, указанные в `requirements.txt`, и PyTorch (если планируется переобучение модели):
```bash
pip install -r requirements.txt
```

### 2. Запуск локального сервера разработки
Запустите сервер с авто-перезагрузкой:
```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```
API документация Swagger/OpenAPI будет доступна по адресу: `http://localhost:8000/docs`

### 🧪 Запуск тестов
Запустите набор юнит- и интеграционных тестов:
```bash
python -m unittest tests/test_tutor.py
```
*Примечание: При запуске тестов автоматически создается и удаляется изолированная база данных `tutor_test.db`.*
